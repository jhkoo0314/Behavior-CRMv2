# 1. 서비스 개요

## 1.1 서비스 이름

**Behavior-Driven CRM (BD-CRM) v2.1**

## 1.2 이 서비스가 무엇인가?

“영업사원이 병원과 의사를 방문하며 하는 모든 행동을 자동으로 기록하고 분석해

**‘어떻게 하면 영업을 더 잘할 수 있는지’** 알려주는 도구입니다.”

쉽게 말해,

> 영업사원에게는 ‘자동 숙제 정리 노트’
> 
> 
> **관리자에게는 ‘수업 참여도 분석표’**
> 
> 같은 역할을 합니다.
> 

---

# 2. 사용자 정의 (누가 사용하는가?)

### **① 영업사원(MR)**

- 병원 방문, 의사 상담, 자료 전달을 하는 사람들
- 목표: *효율적인 활동, 스트레스 없는 보고, 성과 개선*

### **② 팀장/관리자(MR Manager)**

- 여러 영업사원을 관리하는 역할
- 목표: *팀의 성과를 관리하고 위험 병원/약국을 빨리 찾는 것*

### **③ 본부 관리자(Head Manager)**

- 회사 전체 영업 데이터를 보고 싶음
- 목표: *전략 수립, 제품 성과 관리*

---

# 3. 문제 정의 (왜 필요한가?)

현재 제약 영업의 실제 문제점들은 다음과 같습니다:

### **문제 1. “방문했어요”라고 하지만 실제 행동은 알 수 없음**

- 보고서는 형식적으로 작성됨
- 방문 수는 많지만 성과 없는 경우 많음
- 깊이・질・맥락이 없음

### **문제 2. 팀장이 ‘감(감각)’으로 관리함**

- 데이터가 부족해 정량적인 관리가 어려움
- 위험 병원이나 경쟁사 움직임을 놓침

### **문제 3. 보고가 귀찮아서 정보가 손실됨**

- 모바일에서 보고하기 매우 불편
- 영업직은 현장 중심인데 PC 입력 강요함
- 실제로는 많은 정보가 묻혀버림

### **문제 4. 회사 전체적으로 ‘왜 성과가 나왔는지’ 모름**

- 어떤 행동이 효과적인지 분석 불가
- 좋은 사원의 패턴을 표준화할 수 없음

---

# 4. 서비스 목표 (이 시스템이 해결해야 하는 것)

1. **영업사원의 모든 행동을 자동으로 기록하고 자동으로 요약해줌**
    
    → 보고 스트레스 제거
    
2. **행동 기반 KPI를 자동으로 계산**
    
    → 방문 숫자가 아닌 ‘질’을 평가
    
3. **경쟁사 활동, 위험 병원 등을 자동 감지**
    
    → 관리자에게 즉시 경고
    
4. **영업사원별 행동 패턴(Behavior DNA) 분석**
    
    → 개인 맞춤 코칭 제공
    
5. **팀장/관리자에게 실시간 코칭 신호 제공**
    
    → “이번 주 OO사원 리스크 있음, 다음 행동은 이것!”
    

---

# 5. 핵심 기능 (초등학생도 이해 가능 버전)

### **① 자동 방문 기록 기능**

- 병원 근처에 가면 자동으로 방문 체크됨
- “QR 체크인 같은데 자동으로 되는 느낌”

### **② 음성 보고 자동 요약**

- “오늘 ○○병원에서 이런 이야기를 나눴다”
    
    → 말하면 자동으로 정리됨
    
- 기사 요약하는 것처럼 깔끔하게 요약됨

### **③ 행동 점수 자동 계산**

- “숙제 점수”처럼 행동 강도/다양성/질을 점수로 보여줌

### **④ 위험 병원 경보**

- 의사가 반응이 줄거나 경쟁사 활동이 감지되면
    
    → 빨간 경고 표시
    

### **⑤ 다음 행동 추천 (Next Best Action)**

- “지금 이 병원에는 이런 행동이 효과적!”
- AI가 추천

### **⑥ 영업사원별 행동 DNA 분석**

- 어떤 사원은 관계를 잘함
- 어떤 사원은 전환 행동이 강함
- 어떤 사원은 초기 방문에 강함
    
    → 자동으로 성향 분석

# 6. 상세 기능 명세서 (정통 서비스 기획안 버전)

각 항목을 **요구사항 형식**으로 세부 정의합니다.

---

## 6.1 자동 방문/활동 기록

### ● 기능 설명

GPS, 시간대, 패턴을 기반으로 병원을 방문하면 자동으로 “활동 생성”이 이뤄짐.

### ● 사용자 흐름

1. 사용자가 병원 근처 도달
2. 앱이 자동으로 위치 인식
3. “병원 방문 기록할까요?” 팝업
4. 음성/간단 입력으로 요약
5. 자동 저장

### ● 세부 요구사항

- GPS 반경: 기본 30m
- 병원 데이터베이스 매칭 필요
- 재방문 시 이전 방문 로그와 자연스럽게 연결
- 무음 모드, 방해 금지 모드에서도 작동

---

## 6.2 음성 기반 자동 보고

### ● 기능 설명

사용자가 말한 내용을 구조화된 보고서로 자동 변환함.

### ● 처리 과정

1. 음성 → 텍스트 변환
2. 텍스트 → 상담 내용 키워드 추출
3. 행동 항목분류(교육/프로모션/이슈 해결/처방유도 등)
4. 자동 요약
5. 병원/의사별 누적 데이터에 연결

### ● 요구사항

- 상담 길이 10초~3분 권장
- 전문 용어 자동 인식
- 요약은 200자 이하

---

## 6.3 행동 기반 KPI 엔진

### ● 측정 항목

1. **행동 강도(Intensity)**
    
    = 방문 + 콜 + 메시지 + 자료전달 * 가중치
    
2. **행동 다양성(Diversity)**
    
    = 행동 종류 개수
    
3. **행동 질(Quality)**
    
    = follow-up율 + 의사 반응 + 고민 해결 여부
    
4. **행동→성과 전환율**
    
    = 행동 대비 처방 증가RATE
    

### ● 점수 표준화

0~100점 스케일

지사/팀/회사 단위로 통일화

---

## 6.4 코칭 신호(Coaching Signals)

### ● 유형

- 행동 부족 경보
- 관계 악화 경보
- 경쟁사 등장 경보
- 전환 행동 부족 경보
- 병원 관심도 급하락 경보
- 사원별 취약 행동 경보

### ● 팀장 화면에서 표시

- 우선순위 (High/Medium/Low)
- 팀원별 코칭 사유
- 추천 코칭 문구 자동 표시

---

## 6.5 경쟁사 활동 자동 탐지

### ● 탐지 기준

- 의사 멘트: “요즘 ○○사에서 …”
- 가격/샘플 관련 문의
- 특정 제품선호도 변화
- 상담 내용 패턴에서 경쟁사 단어 등장

### ● 출력 방식

- 병원 카드에 “경쟁사 위험도” 아이콘
- 관리자 대시보드에 히트맵 출력

# 7. UI/UX 상세 설계

아래는 **실제 화면을 기획 문서로 만들 때의 수준**으로 정리합니다.

---

## 7.1 메인 대시보드

### ● 구성 요소

- 오늘 해야 할 행동 5개
- 위험 병원 목록
- 최근 상담 요약
- 행동 점수 변화 그래프
- 경쟁사 활동 신호
- 나의 주간 목표 달성도

### ● 시나리오

사용자가 아침에 앱을 켜면

“오늘 해야 할 Top 5 행동”이 먼저 나오도록 설계.

---

## 7.2 병원 상세 페이지

### 포함 요소

- 병원명, 주소, 담당 의사 리스트
- 최근 방문 기록
- 관심도 변화 그래프
- 경쟁사 위험도
- “다음 행동 추천” 버튼
- 병원 특성 메모(진료과/환자 수/매출 규모)

---

## 7.3 보고 작성 화면(음성 우선)

- 음성 버튼 크게 배치
- “말하면 자동으로 정리해주는 방식”
- 자동요약 + 직접수정 가능
- 말풍선 형태의 요약 결과

---

## 7.4 관리자 대시보드

### 영역 구성

1. 팀원 행동 점수 순위
2. 팀원 위험도 리스트
3. 병원 위험 지도
4. 경쟁사 활동 히트맵
5. 팀 목표 달성 현황

# 8. 데이터 구조 설계 (쉬운 설명 포함)

### 핵심 데이터 테이블

1. **Accounts**
    
    → 병원 정보
    
2. **Contacts**
    
    → 의사/스탭 정보
    
3. **Activities**
    
    → 방문/전화/보고 기록
    
4. **BehaviorScores**
    
    → 자동 계산된 행동 점수
    
5. **CoachingSignals**
    
    → 경고 및 코칭 알림
    
6. **CompetitorSignals**
    
    → 경쟁사 활동 기록
    
7. **Reports**
    
    → 음성보고→자동요약 결과 저장

# 9. 정책 및 운영

### ● 권한 구조(RLS)

- 영업사원: 자신의 병원/활동만 조회
- 팀장: 팀원 병원/활동 전체 조회
- 본부장: 조직 전체 조회
- 민감 데이터는 일부 마스킹 처리

### ● 모바일 우선 정책

- 80~90% 기능은 모바일에서 해결
- 데스크탑은 관리용/분석용

### ● 데이터 보존 정책

- 활동 기록 5년 보존
- 음성 파일은 12개월 후 삭제, 요약 데이터만 유지
